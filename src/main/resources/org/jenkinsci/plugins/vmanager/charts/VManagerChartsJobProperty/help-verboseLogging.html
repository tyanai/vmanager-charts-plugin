When enabled, every [vManager Charts] diagnostic line (REST URLs, request headers, payloads, response routing OIDs, per-session listings, summary counters, configuration source, etc.) is printed to the build's console.

When disabled (the default), only WARNING and error lines appear. Turn this on while reproducing an issue, then turn it back off to keep the console clean.

This flag is always taken from this GUI, even when JSON file from workspace is selected as the chart configuration source.
